# NeuralScript — Digit Recognizer v2

## What's improved over v1

### Model (`train_model.py`)
| Feature | v1 | v2 |
|---|---|---|
| Architecture | 2 Conv blocks + Dense | 3 Conv blocks + BatchNorm + GAP |
| Augmentation | None | Rotation ±8°, Zoom ±8%, Shift ±8% |
| Regularization | Dropout 0.3 | Dropout 0.25/0.4 + BatchNorm |
| Training | Fixed LR, 8 epochs | ReduceLROnPlateau + EarlyStopping |
| Checkpointing | Last epoch | Best val_accuracy epoch |
| Expected accuracy | ~99% | ~99.3–99.5% |

Augmentation specifically fixes the **7→1** and **6→9** errors by training on rotated/shifted variants of every digit.

### Preprocessing (`app.py`)
| Step | v1 | v2 |
|---|---|---|
| Invert | Always | Auto-detect (white paper → invert) |
| Centering | None | Bounding-box crop → MNIST-style 20×20 in 28×28 |
| Stroke | None | Gaussian blur 0.5r to smooth resize artifacts |

Bounding-box centering is the biggest real-world fix — MNIST digits are always centered, but user drawings rarely are.

## Setup

```bash
# 1. Install
pip install -r requirements.txt

# 2. Train the improved model (takes 5–15 min depending on GPU)
python model/train_model.py
#    → saves digit_model_v2.keras

# 3. Run Flask
python app.py
#    → http://localhost:5000
```

If you already have `digit_model.keras` from v1, the app will use it as a fallback — but accuracy will be lower. Retrain with `train_model.py` for best results.
